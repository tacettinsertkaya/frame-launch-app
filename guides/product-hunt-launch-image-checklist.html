Launch checklist

Product Hunt images should explain the product before the first comment.

Use this checklist to create gallery screenshots, social cards, and launch page visuals that make the product value obvious.

The minimum launch image set

  1. A hero image that states the product category and outcome.
  2. A workflow screenshot that shows what the user does first.
  3. A proof screenshot with an output, result, or before/after state.
  4. A privacy, speed, or integration image that answers a likely objection.
  5. A social card for X, LinkedIn, Reddit, and newsletter posts.

Write captions like product positioning

Each image should carry one message. Avoid listing features across every panel. Use the first image for the promise, the next two for proof, and the final image for a practical reason to try the product now.

Design rules for launch galleries

  • Use large type and avoid tiny UI labels as the only explanation.
  • Keep backgrounds consistent so the gallery feels like one set.
  • Show real product states rather than abstract decoration.
  • Create one square or wide social version for distribution.

Fast workflow in Frame Launch

Upload product screenshots, choose Website Hero, Open Graph, or Twitter Card size, write the main caption, then duplicate style across the set. Export one proof image first, check it in a social preview, then export the full ZIP.

For Product Hunt, the gallery is not decoration. It is the first product demo.
